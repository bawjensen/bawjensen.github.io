The Language

"Go"

A Little History

  • Invented at Google
  • Project started in 2007
  • Project went public in November 2009
  • Designed to be concurrent
  • Designed to address certain issues with how existing languages were done (i.e. C++)
  • Designed to scale

Why Google Created a New Language

  • Google is huge
  • Google creates new projects all the time
    • They often see a project evolve from small- to large- to huge-scale
  • Google was tired of the existing languages (Java, C++, Python) and their workarounds for modern problems

Where Go Lives in the World

  • Go is less exact than C++
    • C++ is more feature-ful
    • Go focuses on simplicity and a new approach
  • Go gets more converts from languages such as Ruby or Python than from more "large system" languages such as Java/C++

Where Go Lives in the World

  • Go is inspired heavily by C/C++, built from scratch, and borrows concepts/ideals from languages such as Python/Ruby
  • But:
    • Go isn't "OO" (Object Oriented)
    • Go isn't functional
  • So, Go is an attempt to start over and take a different look at programming in the large scale

More History

  • Invented by Robert Griesemer, Rob Pike, and Ken Thompson, employed at Google
  • Inspired by frustrations with C++ way of doing things
    • Waiting for a 45 minute compilation for minor (but central) fixes
    • Concurrency and simplicity seem to be the main tenets
  • Commonly used for the language's simplicity, and its ease of use for concurrent applications

Quick note on concurrency

  • Moore's Law no longer applies
  • Modern personal computers have a minimum of 2 cores
  • Large computing systems, i.e. servers, have many cores
  • Programs written for such systems should be able to take advantage of all of that power

Technical Aspects

  • As it is inspired by C++, the Go language is typically compiled
  • No support (yet) for lazy evaluation, but can be emulated
  • Go is statically, strongly typed
    • Typically explicitly typed, but can be inferred

Technical Aspects

  • Has support for objects, but not inheritance. They're called structs, and are more simplistic
  • Deals with interfaces, which can be seen as a type of interitence
  • Has garbage collection
  • Has a large standard library
  • Typical error handling system
  • Emphasized support for concurrency

package main

import "fmt"

func main() {
    fmt.Println("counting")

    for i := 0; i < 10; i++ {
        defer fmt.Println(i)
    }

    fmt.Println("done")
}

Code Example v4


package main

import "fmt"

// fib returns a function that returns
// successive Fibonacci numbers.
func fib() func() int {
    a, b := 0, 1
    return func() int {
        a, b = b, a+b
        return a
    }
}

func main() {
    f := fib()
    // Function calls are evaluated left-to-right.
    fmt.Println(f(), f(), f(), f(), f())
}
